## LLM 是怎么训练出来的

> 训练大语言模型（LLM）主要分为两个核心阶段：**预训练（Pre-training）** 和 **后训练（Post-training）**。

---

## 第一部分：构建基础模型

### 一、预训练（Pre-training）

在 LLM 能够生成连贯文本之前，它需要先掌握语言的基本规律，这一过程称为**预训练**。预训练是一个计算密集型任务，涉及大量数据和复杂的模型优化。

#### 步骤 1：数据收集与预处理

训练 LLM 的第一步是收集海量高质量的文本数据，目标是构建一个多样化且覆盖面广的数据集，以便模型能够学习到丰富的语言知识和上下文关系。

一个常见的数据来源是 **Common Crawl**——一个免费开放的网页爬取数据库，包含了过去 18 年间约 2500 亿个网页的数据。然而，原始网页数据通常包含大量噪声（如垃圾信息、重复内容和低质量文本），因此数据预处理是必不可少的环节。

> 🔗 [Common Crawl 官网](https://commoncrawl.org/)

#### 步骤 2：分词（Tokenization）

在神经网络能够处理文本之前，文本需要被转换为数值形式，这一过程称为**分词（Tokenization）**。分词将单词、子词或字符映射为唯一的数值 token，这些 token 是语言模型的基本构建单元。

以 GPT-4 为例，其词汇表包含约 **100,277** 个 token，每个 token 对应一个唯一的数值 ID，模型通过这些 ID 来识别和处理文本。

> 🔗 可通过 [Tiktokenizer](https://tiktokenizer.vercel.app/) 直观体验分词过程——输入任意文本，即可查看其如何被拆解为 token 及对应的数值 ID。

#### 步骤 3：神经网络训练（Neural Network Training）

文本经过分词处理后，神经网络的任务是学习**根据上下文预测下一个 token**。模型接收输入 token 序列（例如"我正在烹饪"），经过其复杂的数学结构（即模型架构）处理后，输出对下一个 token 的概率预测。

神经网络主要由两个关键部分构成：

| 组成部分 | 说明 |
| --- | --- |
| **参数（Weights）** | 通过训练学习得到的数值，用于调整模型的行为 |
| **架构（Architecture）** | 定义输入 token 如何被处理以生成输出的数学结构 |

在训练初期，模型的预测几乎是随机的。随着训练推进，当模型正确预测了某个 token 时，系统会通过**反向传播（Backpropagation）** 调整其数十亿个参数，提高正确预测的概率，降低错误预测的概率。这一过程在海量数据集上重复数十亿次，直到模型能够高效捕捉语言规律。

#### 基础模型（Base Model）：预训练的成果

经过预训练，基础模型已学会：

- **理解单词、短语和句子之间的关联**
- **识别训练数据中的统计模式**

然而，基础模型并未针对具体任务优化。可以将其视为一个高级的"自动补全系统"——能够基于概率预测下一个 token，但缺乏对指令的精确理解和执行能力。基础模型有时会逐字复述训练数据，可通过**上下文学习（In-Context Learning）** 进行特定应用，但仍需进一步的训练和优化。

---

### 二、后训练（Post-training）

基础模型虽具备语言理解和生成能力，但尚未精细打磨。为使模型在实际应用中更加实用、可靠和安全，需要进行**后训练（Post-Training）**。这一阶段通常在更小、更专业的数据集上进行微调，以优化模型的行为和性能。

由于神经网络无法像传统软件那样通过显式编程来调整，我们只能通过训练来"编程"它——让模型学习结构化的、带标注的数据集，这些数据集代表了理想的交互示例。

#### 后训练的方式

| 方式 | 目标 |
| --- | --- |
| **指令/对话微调（Instruction Fine-Tuning）** | 让模型学会遵循指令、执行任务、进行多轮对话、遵守安全规范及拒绝恶意请求 |
| **领域特定微调（Domain-Specific Fine-Tuning）** | 使模型适应特定领域需求，如医学、法律、编程等 |

例如，OpenAI 的 InstructGPT（2022）项目聘请了约 40 名人工标注者创建高质量数据集，编写提示（Prompts）并提供符合安全指南的理想响应。如今，许多数据集由 AI 自动生成，再经人工审核和编辑以确保质量。

#### 特殊 Token 的引入

在后训练阶段，还会引入一些特殊 token，帮助模型理解交互的结构：

1. 标记**用户输入**的起始与结束
2. 标记 **AI 响应**的起始位置

这些特殊 token 确保模型能够正确区分提示（Prompt）和回答（Reply），从而生成更符合上下文的响应。

---

### 三、推理（Inference）——模型如何生成新文本

推理是模型生成新文本的过程，可在任何阶段进行，甚至可在预训练中途执行以评估模型的学习效果。

模型接收输入 token 后，会根据训练中学到的模式，为所有可能的下一个 token 分配概率。模型并非总是选择概率最高的 token，而是通过**概率分布采样**来决定下一个 token——类似于抛掷带偏向的硬币，高概率 token 更有可能被选中，但低概率 token 仍有机会被选择。

由于 token 选择具有一定随机性，即使输入相同，模型也可能生成不同的输出。通过这种方式，模型能够生成训练数据中未直接出现但符合统计规律的文本。

---

### 四、幻觉（Hallucinations）——当 LLM 生成错误信息

**幻觉（Hallucination）** 是指 LLM 生成虚假或错误信息的情况。其根本原因在于，LLM 并不"理解"事实——它只是根据训练数据预测最可能的单词序列。

在早期，LLM 的幻觉问题尤为严重：如果训练数据中包含大量类似"谁是…"的问题且都有明确答案，模型可能学习到这类查询总该有一个自信的回答，即使它实际上并不具备相关知识。

#### 减少幻觉的方法

**方法一：训练模型说"我不知道"**

提高模型的事实准确性，需要明确训练它识别自身知识的边界，通常通过**自我询问（Self-Interrogation）** 来实现：

- 由另一个 AI 模型自动生成问题以探测知识盲点
- 若模型生成了错误答案，系统加入新的训练示例，正确回应为"我不确定。能否提供更多上下文？"

训练机制：
- 若模型多次遇到某个问题，它会为正确答案分配较高概率
- 若模型从未遇到过某问题，则会在多个可能的 token 间均匀分配概率，使输出更加随机

经过微调后，模型被显式训练以处理低置信度的输出，例如 ChatGPT-4o 在遇到无意义问题时会正确回应："我不确定那是谁。能否提供更多上下文？"

**方法二：引入网络搜索**

赋予模型访问外部搜索工具的能力，扩展其知识范围，使其超越训练数据的限制：

- 当模型检测到不确定性时，触发一次网络搜索
- 搜索结果被插入到模型的**上下文窗口**中，成为其"工作记忆"的一部分
- 模型在生成响应时参考这些新信息

这种方法不仅减少了幻觉，还显著提升了模型提供准确、实时信息的能力。

---

### 五、模糊记忆与工作记忆

LLM 通常通过两种方式访问知识：

| 记忆类型 | 描述 |
| --- | --- |
| **模糊记忆** | 预训练过程中存储的知识，基于从海量互联网数据中学到的统计模式。覆盖范围广但不精确，无法直接搜索 |
| **工作记忆** | 推理过程中可直接访问的信息，存储在上下文窗口中。通过在上下文中添加相关事实，可显著提高响应质量 |

---

### 六、自我认知

当被问到"你是谁？"或"是什么构建了你？"等问题时，LLM 会根据训练数据生成统计上最可能的回应，除非被显式编程以提供准确答案。LLM 并不具备真正的自我意识，其回应完全依赖于训练中学到的模式。

为了让模型表现出一致的身份，可以使用**系统提示（System Prompt）**——通过预定义的指令描述模型的身份、能力以及局限性，从而引导模型生成符合预期的回答。

## 第二部分：强化学习（Reinforcement Learning）

### 回顾：LLM 训练的三个阶段

| 阶段 | 说明 |
| --- | --- |
| **预训练** | 从海量数据集中学习，形成基础模型 |
| **监督微调（SFT）** | 通过精心挑选的示例优化模型，使其更实用 |
| **强化学习（RL）** | 允许模型从自身经验中学习，持续优化策略 |

尽管预训练和 SFT 已是成熟方法，RL 仍在不断发展，并成为训练流程中不可或缺的一部分。

---

### 一、RL 的目的

人类和 LLM 处理信息的方式存在显著差异。例如，基本算术对人类而言是直观的，而 LLM 则将文本视为一串 token 序列，这对它们来说并不直观——然而，LLM 能够在复杂主题上生成专家级回答，仅仅因为训练过程中见过足够多的示例。

这种认知差异使得人类注释者难以提供一组"完美"的标签来持续引导 LLM 找到正确答案。RL 弥补了这一差距：模型不再仅依赖显式标签，而是通过探索不同的 token 序列并根据哪些输出最有用来获得反馈（**奖励信号**）。随着时间推移，模型学会了更好地与人类意图对齐。

---

### 二、RL 背后的直觉

LLM 本质上是随机的——即使相同的提示，输出也可能不同。我们可以利用这种随机性，**并行生成成千上万甚至数百万个可能的响应**，探索不同路径（有些好、有些差），然后鼓励模型更多地选择较好的路径。

与监督微调（SFT）的对比：

- **SFT**：依赖人类专家提供的标签数据，关注**复制**
- **RL**：允许模型从自身的学习中进步，通过发现哪些响应最有效并在每个训练步骤后更新参数，关注**探索与优化**

---

### 三、RL 并非新事物：AlphaGo（2016）

RL 强大力量的经典例子是 DeepMind 的 **AlphaGo**，它是第一个击败职业围棋选手并最终超越人类水平的 AI。

在 2016 年《自然》杂志的论文中：

- **仅用 SFT 训练**：模型能达到人类水平，但永远无法超越——SFT 关注复制而非创新，无法让模型发现超越人类知识的新策略
- **加入 RL 训练**：AlphaGo 能与自己对弈，改进策略，最终**超越人类专业水平**

RL 使模型能够探索超越人类想象的策略，从而优化其思维和决策能力。

---

### 四、RL 基础概念

| 概念 | 说明 |
| --- | --- |
| **Agent（智能体）** | 学习的主体，负责在环境中采取行动 |
| **Environment（环境）** | 智能体交互的外部世界，根据行动给出反馈 |
| **State（状态）** | 环境在某一时刻的具体情况，智能体据此决定行动 |
| **Action（动作）** | 智能体在某状态下采取的具体行为 |
| **Reward（奖励）** | 数值形式的反馈，正奖励鼓励重复行为，负奖励起抑制作用 |
| **Policy（策略）** | 智能体的决策规则，定义在给定状态下选择不同动作的概率 πθ(a\|s) |
| **Value Function（价值函数）** | 评估某状态的好坏，考虑长期期望奖励 |

通过不断从不同状态和动作中收集反馈，智能体逐渐学习出最佳策略，以在长期内最大化累积奖励。

---

### 五、Actor-Critic 架构

Actor-Critic 是一种流行的强化学习框架，结合了两个关键组件：

| 组件 | 职责 |
| --- | --- |
| **Actor（演员）** | 学习和更新策略 πθ，决定在每个状态下应采取哪个动作 |
| **Critic（评论者）** | 评估价值函数 V(s)，为 Actor 提供反馈，告知其选择的动作是否带来了好的结果 |

**工作原理：**

1. Actor 基于当前策略选择一个动作
2. Critic 评估结果（奖励 + 下一个状态）并更新其价值估计
3. Critic 的反馈帮助 Actor 优化策略，使未来动作能获得更高的奖励

**在 LLM 上下文中的对应关系：**

| RL 概念 | LLM 对应 |
| --- | --- |
| 状态 | 当前文本（提示或对话） |
| 动作 | 生成的下一个 token |
| 奖励 | 奖励模型（例如来自人类反馈） |
| 策略 | 模型选择下一个 token 的规则 |
| 价值函数 | 评估当前文本上下文对最终生成高质量响应的贡献 |

---

### 六、DeepSeek-R1：RL 的实践案例

DeepSeek-R1 是一种在保持开源的同时达到顶尖性能的推理模型，展示了 RL 在 LLM 训练中的强大潜力。该研究提出了两个模型：

| 模型 | 特点 |
| --- | --- |
| **DeepSeek-R1-Zero** | 仅通过大规模 RL 进行训练，跳过了监督微调（SFT） |
| **DeepSeek-R1** | 在 R1-Zero 基础上进一步优化，解决了训练中遇到的挑战 |

#### 1. RL 算法：GRPO

**GRPO（Group Relative Policy Optimisation，群组相关策略优化）** 是 PPO（Proximal Policy Optimisation，近端策略优化）的变种，于 2024 年 2 月在《DeepSeekMath》论文中提出。

**为什么选择 GRPO 而不是 PPO？**

| PPO 的问题 | 描述 |
| --- | --- |
| 依赖评论者模型 | 需要单独的 Critic Model，增加内存和计算开销 |
| 训练复杂性 | 在细致或主观任务中评论者模型可能变得复杂 |
| 高计算成本 | RL 流水线需要大量资源来评估和优化响应 |
| 绝对奖励评估 | 依赖单一标准判断好坏，难以捕捉开放性任务的细微差别 |

**GRPO 如何解决这些挑战？**

GRPO 通过**相对评估**消除了对评论者模型的依赖——响应在一个组内进行比较，而不是通过固定标准判断（类似于学生之间互相比较答案，而非由老师单独评分）。随着时间推移，模型的表现趋向于更高质量。

**GRPO 的训练过程：**

1. **收集数据**：旧策略（模型的旧快照）为每个查询生成多个候选答案
2. **分配奖励**：对组内的每个响应进行评分
3. **计算 GRPO 损失**：测量新策略生成过去响应的可能性，评估相对质量，应用裁剪以防止极端更新
4. **反向传播 + 梯度下降**：计算每个参数对损失的贡献并更新参数
5. **更新旧策略**：偶尔更新旧策略以与新策略匹配，为下一轮比较刷新基准

#### 2. 思维链（Chain of Thought，CoT）

传统的 LLM 训练流程是：预训练 → SFT → RL。DeepSeek-R1-Zero 跳过了 SFT，允许模型直接探索**思维链（CoT）推理**。

CoT 使模型能够像人类一样将复杂问题分解为中间步骤，从而增强推理能力。OpenAI 的 o1 模型也利用了这一点——o1 的表现随着更多 RL 训练和推理时间的增加而持续提升。

DeepSeek-R1-Zero 表现出**反思性倾向**，能够自我精炼推理过程：

- 随着训练推进，模型思考深度增加，生成更长、更详细且更优的响应
- 在没有显式编程的情况下，模型开始重新审视过去的推理步骤，显著提高了准确性
- 模型还经历了"**啊哈时刻（Aha Moment）**"——展示了 RL 如何催生出意想不到且复杂的涌现能力

> **注意**：OpenAI 并未在 o1 模型中展示完整的推理链条，原因是担心**蒸馏风险（Distillation Risk）**——有人可能试图通过模仿这些推理痕迹来恢复大量推理性能。因此 o1 只会总结思维链条，而非完整展示推理过程。

### 七、RLHF（基于人类反馈的强化学习）

对于具有明确可验证输出的任务（如数学问题或事实问答），AI 的回答可以轻松评估。然而，对于**总结、创意写作**等没有单一"正确"答案的领域，如何评估模型的表现？这就是 **RLHF（Reinforcement Learning with Human Feedback）** 的作用所在。

#### 核心思路

完全依赖人工评估不现实（若需要十亿次人工评估，成本极高且效率低下）。更智能的解决方案是**训练一个 AI 奖励模型**，让它学习人类的偏好，从而大幅减少人工工作量。

**为什么使用排名而非绝对评分？**

对响应进行排名比直接评分更容易且更直观——人类更容易判断哪个回答更好，而不是为每个回答分配一个具体的分数。

#### RLHF 的优缺点

| | 说明 |
| --- | --- |
| ✅ **广泛适用性** | 可应用于任何领域，包括创意写作、诗歌、总结等开放性任务 |
| ✅ **简化评估** | 对输出进行排名比生成人工标签或创意输出更容易 |
| ⚠️ **奖励模型局限** | 奖励模型是近似的，可能无法完美反映人类偏好 |
| ⚠️ **奖励黑客风险** | RL 模型可能利用奖励模型的漏洞，生成荒谬但高分的输出（尤其在训练时间过长时） |

#### RLHF 与传统 RL 的区别

| 类型 | 适用场景 | 特点 |
| --- | --- | --- |
| **传统 RL** | 可验证领域（数学、编程） | 模型可无限运行并发现新策略 |
| **RLHF** | 开放性、主观性任务 | 更像微调步骤，用于将模型与人类偏好对齐，而非发现全新策略 |

通过 RLHF，模型能够在开放性任务中生成更符合人类期望的响应，同时减少对人工评估的依赖。然而，奖励模型的局限性仍需谨慎处理，以避免模型生成低质量或不合逻辑的输出。
